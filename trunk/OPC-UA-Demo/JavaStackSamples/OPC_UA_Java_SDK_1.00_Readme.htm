UA Java SDK 1.00.232.0 (Beta) Readme
March 26, 2009

Overview

The UA Java SDK is a set of interfaces, libraries and executables that allow developers to create UA applications in the Java programming environment.
This release of the Java SDK includes:

  • Implementations of  the UA Native Binary stack profile;
  • Simple Client and Server Applications;

The sample applications are available with source code.

Release Notes
Build 232.0 is the first release made available to all members.This release of the SDK supports the released versions of Parts 1 through 8 of the UA Specification.  

 The UA Specification is now available on the OPC Foundation website.

Users should use Mantis to report any issues/bugs.

This will also allow users to find issues identified by others.

Release Process
The release process is designed to balance the needs of different developers in different stages of development. The different stages are described in the following table:

Snapshot Snapshot builds are only minimally tested and are used to give developers quick access to new features and bug fixes. There may be backward compatibility issues and some features may be broken. Developers should not migrate to a snapshot build unless they need the feature and/or bug fixes.
Testing Testing builds are put through basic tests to ensure that nothing is broken but have not been tested for backward compatibility. All Stable and Final builds are posted first as Testing builds which gives developers a chance to check for issues which can be corrected quickly. Any changes will incorporated into a new revision and posted immediately. If no issues come up the last revision of a Testing build will be re-labeled as Stable or Final. Developers are encouraged to look at Testing builds and see if there are issues. Developers should not migrate to Testing builds unless they absolutely need the functionality or are participating in the testing process.
Stable Stable builds are put through basic tests to ensure that nothing is broken but have not been tested for backward compatibility. Stable builds are posted to give vendors a chance to verify that it works with existing products. Vendors who have products under development should migrate to stable builds as quickly as possible. If a vendor is already distributing products they may wish to use the sample applications to test for compatibility problems.
Final Final builds are stable builds that have been verified by one or more vendors. Patches required to fix issues discovered while testing the stable build are applied but no other features are added. Vendors who have products under development should migrate to stable builds as quickly as possible.

Each build is assigned a unique build number that may be qualified with a revision number.
The revision number indicates that patches were applied to a build.
Patches may be created to fix critical issues in final builds even if a newer build is available.
Vendors will need to upgrade to the newest version if they need fixes for stable or snapshot builds.
All files posted to the website or SharePoint will have the build stage and build number in the file name. For example: OPC UA Java SDK 1.00 Redistributable [232.0 Stable].zip
 The major version of the SDK always matches the major version of the latest specification that it supports.
The minor version is incremented each time new features are added after release.
If a previously posted build needs to be patched the revision number will be incremented.
For example, 218.1 is a patch applied to 218.0.

Known Issues
These features are not included in the current release of the SDK: 
  • XML Encoding
  • SOAP / Web services
  • Complete sample applications
  • Client Library (Subscription, MonitoredItem, NodeCache)
  • Server Library (NodeManager, DataSource, EventSource, DataHistorian, EventHistorian)
  • Discovery Server
  • IPv6 not supported.

Installation

 The minimum Java requirement is Sun Microsystems Java 1.6.UA Java SDK has dependencies to following Java Libraries:
  • Log4J (1.2.15) - http://logging.apache.org/log4j/
  • Bouncy Castle Crypto API (1.40) - http://www.bouncycastle.org/

Steps for developers that already have a Java development environment:
  • Unzip OpcUa-xxx-SDK.zip
  • Include OpcUa-xxx.jar to your java project
  • Include dependencies to your java project (See above)
  • Unzip OpcUa-xxx-Examples.zip
  • Run Example1.java

Steps for developers that do not have a Java development environment:
  • Download and Install Java 6.0 Update 13 JDK at http://java.sun.com/
  • Download and unzip Eclipse IDE for Java Developers at http://www.eclipse.org/downloads/
  • After welcome screen, select Java perspective (upper right hand corner)
  • Create new Project "UA-Samples"
        * File / New / Java Project
  • Add Jars OpcUa-XXX.jar, bcprov-jdk14-140.jar and log4j-1.2.15.jar to the project.
        * Drag'n'drop OpcUa-XXX.jar and OpcUa-232.zip over "UA-Examples" in Eclipse.
        * Unzip OpcUa-XXX-libs.zip.
        * Drag'n'drop bcprov-jdk14-140.jar and log4j-1.2.15.jar over "UA-Examples" in Eclipse.
        * Select properties of "UA-Examples" in context menu. Press the button "Java Build Path / Libraries / Add JARs" and select the jars.   
  • Add the examples sources to the project.
        * Unzip OpcUa-XXX-Samples.zip and Drag'n'drop "org"-folder over "src"-folder in Eclipse workspace.
  • Run the examples.
        * ClientExample1 and ClientExample2 are to be run with ServerExamples1.
        * ClientExample3 and ClientExample4 with UA Sample Server of the UA SDK. 
 


Sample Applications

There are 1 server and 4 client sample applications bundled with the SDK.
The samples only implement a single Test service.
The ClientExample1 and ClientExample2 are to be run with ServerExamples1.
The ClientExample3 and ClientExample4 with UA Sample Server of the UA SDK.
Note that the client certificate (ClientCert.der) must be installed in the trusted certificate store used by the UA Sample Server before connecting.
